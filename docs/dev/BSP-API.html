BSP functionality

Each BSP must provide:

  • data import filters ( data, AC config, flight test info )
    • fill var labels display/Tex fields
  • tree state after import (embedded in FDS)

Optional BSP functions:

  • event import filter, including a standard event type list

  • custom plot definitions

  • custom plot data manipulation functions

  • export definitions

  • initial plot after import

  • provide information about the ARMED system state (time limits, which channel it is encoded in) - this can be used to easily remove unnecessary regions at the start and end of the data set.

  • provide FLIGHT MODE channel information.

  • provide flight coordinates (Lat/Lon/Alt) for MAP display

  • provide Google Maps API key for MAP display

  • provide data channels for flight path reconstruction

  • AC model for local playback

  • MAVlink data playback to external visuals

  • List of custom tabs available

BSP required API

The function calls the BSP needs to be able to execute.
